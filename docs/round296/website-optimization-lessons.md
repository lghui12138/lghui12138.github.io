# Round296 Website Optimization Lessons

Version target: `round296-website-optimization-ledger-20260614`

Round296 should turn Round295 release pain into stricter, visible gates. The core lesson is that a local pass is not enough: the public shell, source app, browser evidence, and blocker language must all line up before a release is called current.

## Public Shell Freshness

- Check every public shell HTML entry for the current release marker, not only the home page.
- Keep the public shell repository HEAD, upstream branch, remote branch, and `site-updates.json[0]` in the same evidence chain.
- A stale `edge_refresh` marker is a release blocker, even if the source app itself is current.

## lghui.top Proof

- Public proof must name the real public surface: `lghui.top`.
- Browser and monitor evidence should include the visible entry pages, `site-updates.json`, and expected release marker.
- Real account proof is a separate budget. If account env is absent, write blocked/not-run instead of implying recovery.

## Real Exam Source Truth

- Keep 316/316 source atoms, 61/61 grouped sections, and 201 grouped web question IDs as non-waivable counts.
- Some original short-answer rows have 5 independent items. The UI and JSON should preserve that natural count rather than merging them into fewer answers.
- Source-original comparison should lead copy changes: first prove the original row, then edit display text.

## Private Video Boundary

- Static and mock gates can prove UI safety and API shape, but they cannot prove production storage-backed management.
- Production list, publish, archive, and delete proof requires both real teacher browser proof and `FM_PRIVATE_MEDIA` R2 binding.
- Keep blocked status visible in teacher UI, runbook, release report, and JSON gates.

## 181103 Path-Safe Indexing

- The protected download-folder inventory should publish safe metadata only: counts, type buckets, hashes, route coverage, and review-task coverage.
- Do not publish local paths, raw private routes, or raw source downloads.
- Keep 38 materials, at least 30 study routes, and 61 review tasks wired into the release bundle.

## Roadmap Integration

- Main thread should merge this as Round296 route material, not as a release by itself.
- Round296 acceptance should mention public shell all-HTML freshness, lghui.top proof, real-exam source row counts, private-video R2 blocker, and 181103 path-safe indexing.
- Later auth rounds should reuse the same proof split: public anonymous proof, real account proof, storage binding proof, and blocked/not-run summaries.
