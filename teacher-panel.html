部署指南

📦 大视频（>100MB）方案

GitHub 仓库单文件限制 100MB。几百 MB 的视频用 GitHub Release 附件：

  1. GitHub 仓库 → Releases → Draft a new release
  2. Tag：video-v1，Title：视频批次 1
  3. 把视频文件拖进 "Attach binaries" 区域（单文件上限 2GB）
  4. 点击 Publish，右键附件复制链接 → 粘贴到上面"视频 URL"

💾 导出部署

点击右上「导出 resources.json」，把文件 commit 到仓库根目录，学生访问 /resources.html 即可看到。

🛡️ 防下载保护

  • 视频播放时带 动态用户水印（用户名 + 时间戳，防录屏溯源）
  • 禁用浏览器下载按钮、右键菜单、画中画、远程回放
  • URL 运行时注入，不暴露在 DOM 初始 HTML
  • 拦截 Ctrl+S / Ctrl+U 快捷键
  • meta noindex + 无 Referer

⚠️ 实话：没有任何前端方案能 100% 阻止录屏和专业工具扒流。GitHub Release 附件是公开 URL，拿到 URL 的人仍可下载文件。最强防护是 B 站嵌入（自带反爬）或付费 DRM 服务。

数据与同步

修改我的登录密码

输入当前密码后设置一个新的登录密码；密码只保存哈希，不会明文保存，也不能改回默认密码或最近用过的旧密码。

建议使用 12 位以上短语；不能复用旧密码或默认密码。
撤销其它登录会话

输入当前密码后，让之前在其它浏览器或设备上签发的教师登录立即失效；当前浏览器会自动换新会话继续使用。

用于发现异常登录后立即踢下线。
导入学生数据快照

从学生设备导出的 JSON 只作为本机 localStorage 离线参考，不能并入教师监控累计真值。格式为 FMAnalytics.exportSnapshot() 的输出。

导出当前聚合数据

下载当前可见数据 CSV；未持久化或本机离线字段不会写成服务端累计真值。

导出 JSON 快照

服务端 summary 会按服务端来源导出；本机 localStorage 快照只作离线备份。

清空本设备分析数据

清除 sessions/scores/progress 三类记录。不可恢复。登录账户不受影响。
